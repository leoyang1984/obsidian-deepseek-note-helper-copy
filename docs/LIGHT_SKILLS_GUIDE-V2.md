# Obsidian DeepSeek 插件 - “轻 SKILL” 使用指南

欢迎体验完全基于 Obsidian 哲学（“一切皆本地纯文本”）打造的 **Markdown 轻 SKILL 系统**！

借助这个系统，您不需要编写任何复杂的代码，只需要像写普通笔记一样写一段带 YAML 头的 Markdown 文本，就能瞬间为您打造出专属的 AI 工作流快捷命令。

---

## 🚀 什么是“轻 SKILL”？

它是一个**将您的常用 Prompt 固化为 Obsidian 原生命令**的系统。
当您执行 Skill 时，插件会自动抓取您当前的“上下文”（如选中的文字、当前笔记标题等），将它们无缝注入到您的 Prompt 模板中，并直接发送给大模型处理。

### 核心优势
- **零代码**：纯 Markdown 编写。
- **热重载 (Hot Reload)**：只要您保存了文件，改动（包含名字、Prompt 等）瞬间生效，无需重启 Obsidian。
- **极致便携**：看到别人分享的优质 Prompt？复制粘贴成一篇笔记，它就变成了一个鲜活的工具！

---

## 🛠️ 如何创建一个轻 SKILL？

1. **设置技能目录**：
   在 DeepSeek 插件的设置面板中，找到 `Skills directory` 选项（默认值为 `DeepSeek-Skills`）。确保您的库根目录下有这个文件夹。
2. **新建笔记**：
   在上述文件夹中新建一篇普通的 Markdown 笔记，文件名随意。
3. **编写 YAML 头部与模板**：
   在文章最开头写上配置参数，在横线下方写上您的 Prompt 咒语。保存后，一个新命令就诞生了！

---

## 📖 变量字典：支持注入的魔法占位符 `{{}}`

在您的 Prompt 正文（YAML 横线下方区域）中，您可以使用以下占位符，插件会在执行时**自动将它们替换为当前编辑器的真实状态**：

| 占位符变量 | 描述说明 | 最佳适用场景 |
| :--- | :--- | :--- |
| `{{selection}}` | **当前选中的文本**。如果执行命令时没有选中任何文字，该变量会被替换为空。 | 文本润色、翻译、总结、解释专业名词等针对局部文本的操作。 |
| `{{title}}` | **当前激活的笔记标题**（不包含 .md 后缀）。 | 让 AI 知道当前的宏观语境，有助于提高生成的准确性。 |
| `{{content}}` | **当前激活笔记的全文内容**。 | 总结全文提纲、根据全文内容生成标签、或对长文进行整体评价。 |
| `{{clipboard}}` | **系统剪贴板中复制的最新文本**。 | 将外部浏览器、微信或其它软件中复制的网页/聊天记录，快速和笔记内容结合在一起喂给 AI 处理。 |

*(注：随着版本迭代，未来将支持更多变量，如结合日期的魔法变量，敬请期待！)*

## ✨ 配置参数说明 (YAML 前言)

每个 Skill 文件的最顶部必须包含一段由 `---` 包裹的 YAML 头部控制台：

```yaml
---
name: "在命令面板中显示的名称"
action: "to_chat" 
---
```

- **`name`** (必填)：这个名称将直接注册到 Obsidian 的命令面板里。当你按下 `Cmd/Ctrl + P` 搜索 `Skill: [您的name]` 就能找到它。
- **`action`** (必填)：指定大模型处理完后如何输出。目前由于安全和体验考虑，仅开放 `to_chat` 模式（即将组装好的 Prompt 发送至侧边栏聊天框并自动执行多轮对话）。未来将开放 `replace`（原位替换）等高阶动作。

## 📋 抄作业专区：开箱即用的优质 SKILL

您可以直接将以下内容复制并保存到 `DeepSeek-Skills` 文件夹的 `.md` 文件中，马上体验！

### 案例 1：沉浸式双向翻译 (中英互译)
对于外语学习者或重度阅读者，自动判断语种并翻译。

> 新建文件: `神级翻译.md`
```yaml
---
name: "神级双向翻译"
action: "to_chat"
---
你是一个精通跨文化传播的顶级翻译专家。请帮我翻译下面这段文本：
如果它是中文，请翻译成地道、符合 native speaker 习惯的英文；
如果它是英文，请翻译成信达雅、符合中文阅读习惯的中文。

请注意当前文章的标题是《{{title}}》，翻译时可以参考这个大的语境。
不要输出任何解释性话语，直接返回翻译结果：
---
{{selection}}
```

### 案例 2：费曼技巧卡片 (复杂概念降维打击)
当你在读一篇艰涩的学术笔记时，划词召唤 AI 解答。

> 新建文件: `费曼概念解析.md`
```yaml
---
name: "费曼概念解析"
action: "to_chat"
---
我正在阅读笔记《{{title}}》。文中有一段内容我不太理解，请帮我用“费曼技巧”对下面这段选中的内容进行降维打击式解释。

要求：
1. 请用大白话解释它的核心概念，假设我是一个只有高中文化基础的外行。
2. 必须举一个生活中极其生动、甚至有些搞笑的例子来类比它。
3. 提取出这部分内容的 3 个核心 Keywords。

待解释的晦涩内容：
---
{{selection}}
```

### 案例 3：暴躁代码 Reviewer (程序员快乐型)
帮你检查代码的逻辑漏洞，并用毒舌的方式给出建议。

> 新建文件: `毒舌代码审查.md`
```yaml
---
name: "毒舌代码审查"
action: "to_chat"
---
你现在是一个拥有 20 年经验、脾气极其暴躁但技术顶尖的资深架构师。
我现在正在写一篇名为《{{title}}》的开发文档。我选中了一段我刚写的代码或技术方案，请你以最挑剔的眼光帮我 Review。

要求：
1. 毫不留情地指出哪里写法冗余、有没有潜在的 Bug、时间/空间复杂度怎么优化。
2. 给出你心里那个完美的重构后代码。
3. 语气要高冷、毒舌但句句切中要害。

我选中的破代码/方案如下：
---
{{selection}}
```

## 🔮 未来计划：魔法变量开发路线图

轻 SKILL 系统的潜力远不止于目前的四个基础变量。得益于 Obsidian 极度开放的纯本地 API，我们能在未来解锁更具想象力的变量注入体系。

以下是我们将按**实现难度或复杂度**（由低到高）逐步推进的变量清单：

### 🏁 第一阶段（极容易实现）：基础环境系
*预计下一小版本更新即可实装，直接通过系统时间与原生 API 提供。*
- **`{{date}}` / `{{time}}`**：执行命令时的系统当前日期和时间。非常适合放在“自动新建卡片并打上时间戳章”等场景。
- **`{{path}}` / `{{folder}}`**：当前文件所在的绝对路径或外层文件夹的名字，方便大模型知晓你的分类笔记所处的领域上下文。

### 🚀 第二阶段（中等难度）：元数据与属性系 (Properties)
*需要单独解析 YAML 头部数据，并对缺失属性做防错处理，这是高频的刚需。*
- **`{{prop:属性名}}` / `{{yaml:属性名}}`**：精确提取当前笔记 yaml 头部里的特定键值对（例如 `{{prop:author}}` 或 `{{prop:status}}`）。能够做到极其细粒度的指令控制。
- **`{{tags}}`**：直接提取当前笔记已被打上的所有标签（如 `#AI #效率`），让模型能够结合已有的业务标签推断是否要扩写新内容。

### 🧠 第三阶段（高阶应用）：双链关系与全局视野
*需要调用复杂的 Obsidian 缓存 API 并进行图谱数据的遍历。*
- **`{{outlinks}}`**：自动抓取当前笔记正文中引用的所有内部链接（出链）列表。大模型可借此获取周边线索和知识网路。
- **`{{backlinks}}`**：提取所有引用了当前这篇笔记的其他文件列表（反向链接）。
- **`{{folder_files:文件夹路径}}`**：动态抓取某个指定文件夹下的所有文件名列表（例如：`{{folder_files:/读书笔记}}`）。可以实现夸文档的比对、查漏补缺等宏大场面。

### 🌀 第四阶段（究极玩法）：执行时动态交互
*需要修改 Obsidian 原生 UI 或使用模态弹窗，执行流最复杂。*
- **`{{prompt:提示语}}`**：动态输入变量。当执行含有这个变量的命令时，屏幕中央会先弹出一个小输入框，用户输入文本（如“薛定谔的猫”）后，系统再把文本注入。这将把原生的静态 Skill 彻底盘活成可互动的超级工具链！
