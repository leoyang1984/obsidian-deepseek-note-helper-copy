# Obsidian DeepSeek 插件 - “轻技能 (Light Skill)” 全攻略

欢迎进入 **Agentic Workflow（智能体工作流）** 时代！本指南将带你从零开始，掌握如何通过简单的 Markdown 笔记，将 DeepSeek 的强大能力深度集成进你的 Obsidian 笔记库。

---

## 🚀 什么是“轻技能 (Light Skill)”？

它是一个**将您的常用指令固化为 Obsidian 原生命令**的系统。借助 Obsidian “一切皆本地纯文本”的哲学，你不需要编写任何代码，只需写一段带 YAML 头的 Markdown，就能打造出专属 AI 工具。

### 核心优势
- **零代码**：纯 Markdown 编写，像写笔记一样简单。
- **热重载 (Hot Reload)**：保存即生效，无需重启插件。
- **三种颗粒度**：支持**单步指令**、**原地斜杠命令**以及**多步 AI 流水线 (Pipeline)**。

---

## 🪄 三种唤起方式

1.  **命令面板 (`P`)**：按下 `Cmd/Ctrl + P` 搜索技能名称即可触发。
2.  **快捷键**：在 Obsidian 设置中的“快捷键”里，搜索技能名称并绑定任意组合键。
3.  **斜杠命令 (`/ds`) —— 推荐**：
    - 在编辑器输入 `/ds` 即可弹出技能菜单。
    - **智能感应**：在文字末尾输入 `/ds` 会自动抓取当前行；在空行输入则自动抓取上一段话。
    - **一键回车**：在设置中开启“默认技能”后，输入 `/ds` 后直接按回车即可瞬间触发替换。

## 🛠️ 配置说明 (YAML 头部)

每个技能文件的最顶部必须包含 YAML 头部配置：

```yaml
---
name: "技能显示名称"
action: "replace" # 动作：replace(替换), insert_below(插在下方), to_chat(发送到聊天框)
---
这里是你的 Prompt 提示词模板...
```

### 常用变量占位符 `{{}}`：
| 变量 | 描述 |
| :--- | :--- |
| `{{selection}}` | **当前选中的文本**（若无选中，斜杠命令会自动按感应逻辑填充）。 |
| `{{title}}` | 当前笔记的标题。 |
| `{{content}}` | 当前笔记的全文本内容。 |
| `{{clipboard}}` | 系统剪贴板内容。 |

## 🏗️ 进阶：Pipeline 流水线模式

当你的任务需要“AI 先思考 -> 暂停请你修改 -> 你确认后再继续执行”时，请使用 `mode: pipeline`：

```yaml
---
name: "✨ 人机协作润色官"
mode: "pipeline"
---
[STEP: 步骤1_逻辑重构]
action: process
请对以下文字进行深度逻辑梳理：{{selection}}

[STEP: 步骤2_人工确认]
action: ask_user
这是 AI 准备的初稿，请你微调：{{步骤1_逻辑重构}}

[STEP: 步骤3_完成插入]
action: replace
{{步骤2_人工确认}}
```

## 📋 优质案例推荐 (可直接复制使用)

### 案例 1：⚡ 极速润色 (适合斜杠命令)
> 文件名: `Slash Polish.md`
```yaml
---
name: "⚡ 极速润色"
action: "replace"
---
请对以下文字进行专业化重写，保持意思不变，但语言更精炼、地道。
注意：**只返回润色后的正文，严禁输出任何多余的解释。**

待处理：{{selection}}
```

### 案例 2：🔤 神级双向翻译
> 文件名: `Deep Translation.md`
```yaml
---
name: "🔤 神级翻译"
action: "replace"
---
你是一个顶级翻译专家。如果输入是中文则翻译成英文，反之亦然。
只返回翻译结果：{{selection}}
```

### 案例 3：🧠 费曼概念解析 (发送到聊天框)
> 文件名: `Feynman Sense.md`
```yaml
---
name: "🧠 费曼概念解析"
action: "to_chat"
---
请用“费曼技巧”对以下内容进行降维打击式解释。假设我是一个高中生，请举一个生动的例子：
{{selection}}
```
