# Obsidian DeepSeek 插件 - “轻 SKILL” 使用指南 V3

欢迎进入 **Agentic Workflow（智能体工作流）** 时代！V3 版本不仅支持简单的 Prompt 固化，还引入了强大的 **Pipeline（流水线）模式**，让您的 Obsidian 笔记真正变成一个能思考、会对话、懂协作的智能系统。

---

## 🚀 什么是“轻 SKILL”？

它是一个**将您的常用指令固化为 Obsidian 原生命令**的系统。

### V3 的两大进化方向：
1.  **单步直达 (Basic)**：选中文字 -> 运行命令 -> 得到结果（适合写推文、总结等简单任务）。
2.  **多步流水线 (Pipeline)**：AI 先思考 -> 停下来问您 -> 确认后修改正文（适合论文润色、专业报告等复杂任务）。

---

## 🪄 唤起方式：除了快捷键，还有“斜杠”

在 V3 版本中，您不仅可以为技能设置全局快捷键，还可以直接在编辑器中通过斜杠命令唤起：

### 斜杠命令 (`/ds`) —— **推荐交互**
- **操作**：在编辑器中输入 `/ds`（默认触发词，可在设置中修改）。
- **智能感应**：
    - **行模式**：在文字末尾输入 `/ds`，AI 会自动抓取当前行。
    - **段落模式**：在空行输入 `/ds`，AI 会自动向上采集整个段落。
- **一键执行**：如果在设置中指定了“默认技能”，打完 `/ds` 直接敲 `Enter` 即可瞬间触发。

---

在 YAML 头部配置 `mode` 或 `action` 决定了技能的运作方式：

### 1. 聊天窗口模式 (`to_chat`)
AI 处理完后，自动打开侧边栏聊天框，方便您继续深入追问。
- **配置**：`action: to_chat`

### 2. 直接编辑模式 (`replace` / `insert_below`)
AI 生成的内容直接写回您的笔记中。
- **配置**：`action: replace` (替换选中) 或 `action: insert_below` (插在下方)。
- **提示**：此类动作会自动弹窗 Notice 提示 AI 正在思考，完成后直接修改文件。

### 3. 流水线模式 (`mode: pipeline`) —— **V3 核心**
通过串联多个步骤实现复杂逻辑。
- **配置**：`mode: pipeline`
- **语法**：使用 `[STEP: 步骤名]` 来定义一个环节。
- **变量接力**：后一步可以使用 `{{前一步名}}` 来调用前面的 AI 输出。

---

## 🏗️ Pipeline：流水线语法详解

一个典型的 Pipeline 文件结构如下：

```yaml
---
name: "工作流名称"
mode: "pipeline"
---
[STEP: 环节1]
action: process
这是一段处理指令。

[STEP: 环节2]
action: ask_user
我想让你根据 {{环节1}} 再改写一下。
```

### 动作 (Action) 列表：
| 动作 ID | 效果说明 | 适用场景 |
| :--- | :--- | :--- |
| `process` | **后台处理**，不展示给用户。 | 预研、中间步骤。 |
| `ask_user` | **人工干预**，侧边栏弹窗等待用户编辑和确认。 | **核心特性**，人机协作润色。 |
| `to_chat` | 打开侧边栏发送消息。 | 将中间结果汇总展示。 |
| `insert_below` | 将本步结果垂直插入文档下方。 | 最终定稿入库。 |
| `replace` | 替换当前文件选区。 | 直接修改原文。 |

## 📖 变量字典

| 变量 | 描述 |
| :--- | :--- |
| `{{selection}}` | 选中文字 |
| `{{title}}` | 文件名 |
| `{{content}}` | 全文内容 |
| `{{clipboard}}` | 剪贴板 |
| `{{步骤ID}}` | **(Pipeline专属)** 前序步骤的 AI 输出 |

## 🌟 案例：人机协作润色仪

这是一个极具代表性的 V3 技能，展示了如何利用“人工确认”：

```yaml
---
name: "✨ 专业文档润色官"
mode: "pipeline"
---
[STEP: 步骤1_逻辑重构]
action: process
请对以下文字进行深度逻辑梳理和语序调整，让其读起来像《自然》杂志一样专业：
{{selection}}

[STEP: 步骤2_人工确认]
action: ask_user
这是我为您准备的初稿，您可以在此处调整语气，确认后将帮您插入文档：
{{步骤1_逻辑重构}}

[STEP: 步骤3_完成插入]
action: insert_below
---
**润色后的定稿：**
{{步骤2_人工确认}}
---
```

## 🔮 进阶技巧与样式
- **视觉反馈**：Pipeline 执行过程中，右上角会有 Notice 实时显示进度。
- **优雅挂起**：当执行到 `ask_user` 时，插件会贴心地自动打开交互侧边栏，并渲染出一个带有玻璃质感效果的交互卡片。
- **防错机制**：如果您在等待 AI 时切换了笔记，插件在最后插入时会再次确认您的活跃编辑器，防止填错位置。
